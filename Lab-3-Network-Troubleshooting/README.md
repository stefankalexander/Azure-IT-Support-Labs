# Lab 3 — Network Troubleshooting

## Objective
Diagnose network connectivity issues using common command-line tools.

## Tools Used
- Windows Command Prompt
- Azure Virtual Machine

## Commands Executed
ipconfig /all
ping google.com
nslookup google.com
tracert google.com

## Purpose of Each Command

- ipconfig /all — view network adapter configuration
- ping — test connectivity to remote host
- nslookup — verify DNS resolution
- tracert — identify network path to destination

## Example IT Support Scenario

User reports they cannot access the internet.

### Troubleshooting Process

1. Verify IP configuration
2. Test connectivity using ping
3. Check DNS resolution
4. Identify network routing issues

## Outcome

Confirmed network connectivity and demonstrated the troubleshooting workflow used by IT support technicians.
